# AGENTS.md

## セキュリティルール

- 新しいプロジェクトを開始したら `/security-review` を実行する
- 本番デプロイ前に必ず `/security-review` を実行する

## サイト編集ルール

- HTMLの編集元はルート直下のファイルとする
- `public/` 以下はビルドで再生成されるため、直接編集しない
- 編集後は `npm run build` を実行し、`public/` に変更を反映する
- ビルド後に `git diff --check` を実行する
- 既存の未コミット変更や未追跡ファイルを、許可なく削除・上書きしない

## Vercel本番デプロイルール

- デプロイ前に `npx --yes vercel build --prod` で本番ビルドを検証する
- 検証済み成果物をVercelへ本番デプロイする
- デプロイ後、必要に応じて新しいデプロイを本番へpromoteする
- `www.moji-lamcompany.com` が新しいデプロイを参照していることを確認する
- トップページと変更対象ページがHTTP 200を返すことを確認する
- 公開HTMLまたは実画面を確認し、変更が本番へ反映されたことを検証する
- Vercelの一時URLだけを確認して、本番反映完了と判断しない
